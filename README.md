# portal-vip-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定域名或网站，而是作为通用内容存档和快速访问入口。

## 项目简介

portal-vip-notes 是一个轻量级的静态页面集合。仓库中的每个 HTML 文件都是一个独立的页面，可以单独打开或嵌入其他环境中。适合用于记录、展示或归档一些轻量级的信息页面。

## 目录说明

```
portal-vip-notes/
├── index.html          # 主入口页面（可选）
├── pages/              # 存放各个独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/` 目录下存放各个独立页面，每个页面应自包含或引用 `assets/` 中的公共资源。
- `assets/` 用于存放多个页面可复用的样式、脚本或图片文件，避免重复。
- `index.html` 作为可选的主入口，可用于导航或展示页面列表。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面内容应保持简洁、干净，避免嵌入外部跟踪或广告脚本。
- 每个页面建议在文件头部添加注释，简要说明页面用途或归档时间。
- 页面之间无强制关联，可独立更新或删除。

## 维护说明

- 新增页面：在 `pages/` 下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应文件，建议在注释中记录变更。
- 删除页面：移除对应文件，并检查 `index.html` 中的链接（如有）。
- 资源管理：公共资源应放入 `assets/`，避免在每个页面中重复复制。
- 提交信息：建议使用清晰简洁的提交信息，如“新增页面：xxx”或“更新页面：xxx”。

## 许可

本项目内容仅供归档与展示使用。如涉及第三方资源，请遵守相应授权协议。
